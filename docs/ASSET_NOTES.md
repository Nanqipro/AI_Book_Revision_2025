正式扩大仓库公开范围前，建议权利人逐图确认：

1. 原作者或生成者；
2. 原始来源链接或内部项目记录；
3. 允许的使用范围与许可证；
4. 是否需要署名、引用、模型/工具披露或第三方商标说明；
5. 是否含有人脸、账号、学号、订单号、患者数据、通知、二维码或其他个人信息；
6. 是否应替换为团队自制、可再许可或版权状态明确的版本。

第 6 章的部分历史输出截图显示通用开发路径 `E:\\Jupyter\\data`，当前未发现用户名、个人目录或密钥，但后续重新运行实验时仍建议改用相对路径并更新截图。

在完成台账前，不应仅凭“文件已放入 GitHub 仓库”推定这些图片可以复制出版、商业使用或再分发。

## 本地模板遗留物

`attachment/`、`figures/`、`gallery/`、`references/`、`attachment.tex` 与旧 `book.pdf` 没有被当前 `book.tex` 引用，其中部分内容来自上游 LaTeX 模板并含第三方姓名或邮箱。它们已被 `.gitignore` 精确排除，保留在本地供原作者核对，不应通过宽泛的 `git add .` 误传到公开仓库。
